# 05 — Plan stage: Writing Plan + ChromaDB Content & Style Notes

## What to build

The Plan stage takes the Brain Dump and produces a structured Writing Plan, while simultaneously surfacing relevant past thinking from the Second Brain. Claude (via OpenRouter) synthesises the full Brain Dump into a structured outline of 2-8 sections, each with a title and bullet points. In parallel, the shared ChromaDB is queried twice: once for Content Notes (topically relevant to the Brain Dump) and once for Style Notes (prose exemplars that best represent Azeem's rhetorical patterns). Content Notes are shown per section as cards on the right. The user can reorder, delete, and add sections, and pin Content Notes into specific sections before proceeding to Draft generation.

The ChromaDB read from `Notion Second Brain (Code)/data/chroma/` — the existing collection. Nothing is written to it in this stage.

## Acceptance criteria

- [ ] Writing Plan is generated by Claude from the full Brain Dump (via OpenRouter) on stage entry
- [ ] Plan is displayed as collapsible sections, each with a title and bullet point list
- [ ] Sections can be reordered via up/down controls
- [ ] Sections can be deleted
- [ ] A new blank section can be added
- [ ] ChromaDB is queried for Content Notes (topic relevance) and Style Notes (prose exemplars) using the existing `Notion Second Brain (Code)/data/chroma/` collection
- [ ] Content Notes are shown as cards in a right-side panel, with relevance matched per section
- [ ] User can pin a Content Note to a specific section; pinned notes are passed to Draft generation
- [ ] Style Notes are retrieved and stored in `st.session_state` for Style Fingerprint extraction in the Draft stage
- [ ] "Generate Draft →" button stores the approved Writing Plan and navigates to the Draft stage
- [ ] Loading state shown during Writing Plan generation and ChromaDB retrieval
